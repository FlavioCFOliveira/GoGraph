Headline benchmark and load-test numbers for the GoGraph **v0.5.0**
release. v0.5.0 is a **feature release** that adds live, tombstone-aware
`db.*` schema introspection (`db.labels()`, `db.relationshipTypes()`,
`db.propertyKeys()`) and three exported `lpg.Graph` enumeration methods.

algorithm, so there is no performance-relevant delta against
[v0.4.0](v0.4.0.md). The canonical guard band is therefore inherited
unchanged, and the per-change record continues to live in
[`history/`](history/) and [`history/LEDGER.md`](history/LEDGER.md).

## Performance impact of this release

None on the hot path. The change set is confined to:

- **`graph/lpg/introspect.go`** â€” three new enumeration methods
  (`NodeLabelsInUse`, `RelationshipTypesInUse`, `PropertyKeysInUse`)
  that scan live graph state and skip tombstoned elements. They are
  invoked only when a client calls the corresponding `db.*` procedure,
  never during `MATCH`, traversal, or write execution.
- **`cypher/procs/builtin_db.go`** â€” the `db.*` procedures now read
  through `BuiltinSources` closures that delegate to those methods.

Because none of this touches the search, traversal, write, or recovery
paths, the guard band above is the authoritative performance picture for
v0.5.0.
