# GoGraph v1.0.0

The first stable release of GoGraph, a Go module for graph
persistence, manipulation, and fast search that scales from
in-memory graphs to graphs that exceed RAM.

## Highlights

- A unified API surface in `graph/`, `graph/adjlist`, and
  `graph/csr` covering directed and undirected graphs, multi-
  graphs, self-loops, and the labelled property-graph model.
- Durable single-writer transactions with WAL group-commit and
  atomic snapshot directories.
- A Tier 2 mmap-backed CSR file format with `madvise` hints and
  semi-external algorithms that operate without first loading
  the graph into the Go heap.
- The canonical analytical algorithm catalogue: BFS / DFS /
  Dijkstra / Bellman-Ford / A\* / direction-optimising BFS /
  Yen / Floyd-Warshall / Johnson APSP / Tarjan SCC / Hopcroft-
  Tarjan BCC / Hierholzer / Hopcroft-Karp / Hungarian / Dinic
  max-flow / Stoer-Wagner global min-cut / Brandes betweenness
  / PageRank (in-memory, personalised push, semi-external) /
  Leiden / Label propagation.
- Interchange with CSV, GraphML, DOT, and JSON Lines.
- LDBC SNB SF1 / SF10, DIMACS 9 SSSP, and Graph500 RMAT harnesses.

|------------------------------------|----------------------------|
| `Mapper.Intern` (hot key)          | 17 ns/op, 0 allocs         |
| `adjlist.HasEdge` (hot cache)      | 49 ns/op, 0 allocs         |
| `csr.NeighboursByID`               | 10.6 ns/op, 0 allocs       |
| `csr.BuildFromAdjList` over 10^7 E | ~51 ms                     |
| `search.BFS` on 10^7-node chain    | 38 ms, 1.25 MB peak heap   |
| `search.Dijkstra` 1M × 4M graph    | ~320 ms                    |
| `wal.Encode` 4 KiB frame           | 4.9 GB/s                   |

| `csrfile.Reinterpret[uint64]`      | 1.31 ns/op, 0 allocs       |

The full set of measured numbers lives in `docs/profiling.md` and
`docs/optimisations.md`.

## Breaking changes

None. This is the first stable release.

## Known limitations

- Johnson APSP currently restricts to non-negative edge weights;
  the Bellman-Ford reweighting pass for negative edges is
  deferred. Use `Floyd-Warshall` when weights can be negative.
- Yen's k-shortest is O(k * (V + E) log V); Eppstein's algorithm
  (better k > 1000) is deferred.
- Leiden ships the simplified local-moving + connected-component
  split form; the refinement / aggregation phases of the Traag-
  Waltman-van Eck 2019 paper are deferred.
- `adjlist.AddEdge` is dominated by the copy-on-write allocation
  (~423 ns/op on a 1M-node graph); the delta-log in-place atomic-
  append variant is on the v1.1 roadmap.
- `bench/ldbc.Run` only wires the synthetic mode in v1; the LDBC
  Datagen integration is deferred.

## Upgrading

This is the first release; no upgrade path exists yet. The
upcoming v1.1 changes will be documented in `CHANGELOG.md` and
follow the `docs/semver.md` policy.
