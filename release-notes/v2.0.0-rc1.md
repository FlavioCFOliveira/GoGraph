## Overview

v2.0.0-rc1 is the first release candidate for GoGraph v2.0.0. It ships the
full **Cypher execution engine**, a production-ready **Bolt v5 server**,
**90.7 % TCK parser conformance**, a comprehensive benchmark and profiling
harness, and a 4-hour-soak-tested persistence layer.

Every component from Sprint 21 through Sprint 32 (12 sprints) is included.
Nothing from v1.x is removed; this release candidate is backward-compatible
for all public APIs unchanged since v1.2.0.

This tag is a release candidate, not a stable release. The execution-level
TCK pass rate is 10.4 %; v2.0.0 stable requires â‰¥ 80 %. See
[docs/semver.md](../docs/semver.md) for the full release-gate specification.

## Highlights

### Cypher Execution Engine

GoGraph now includes a complete Cypher query engine built on the
[Volcano/iterator model][volcano]. The engine supports:

- **Reading:** `MATCH`, `OPTIONAL MATCH`, `WHERE`, `RETURN`, `WITH`,
  `ORDER BY`, `LIMIT`, `SKIP`, `DISTINCT`, variable-length path patterns.
- **Writing:** `CREATE`, `MERGE`, `SET`, `REMOVE`, `DELETE`, `DETACH DELETE`.
- **Aggregation:** `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`, `COLLECT`,
  `PERCENTILE_CONT`, `PERCENTILE_DISC`, `ST_DEV`.
- **Schema DDL:** `CREATE INDEX`, `DROP INDEX`, `CREATE CONSTRAINT` (UNIQUE,
  NOT NULL), `DROP CONSTRAINT`, `SHOW INDEXES`, `SHOW CONSTRAINTS`.
- **Procedures:** `db.indexes()`, `db.constraints()`, `db.labels()`,
  `db.relationshipTypes()`, `db.propertyKeys()`, `db.schema.visualization()`.
- **Parameters:** `$paramName` binding via `map[string]expr.Value`.
- **Plan cache:** LRU cache with stats-driven invalidation.
- **EXPLAIN / PROFILE:** tree-formatted physical plan output with `db_hits`.

See [docs/cypher.md](../docs/cypher.md) for the full language reference.

### Bolt v5 Server

GoGraph can now act as a drop-in Neo4j-compatible graph server, accepting
connections from `neo4j-go-driver` v5 and `cypher-shell`:

```go
eng := cypher.NewEngine(g)
srv := server.NewServer(eng, server.Options{MaxConnections: 1024})
go srv.ListenAndServe(context.Background(), ":7687")
```

Features:
- Bolt protocol versions 5.0â€“5.6 (Bolt 4.4 fallback).
- Basic authentication and TLS.
- Explicit transactions (`BEGIN` / `COMMIT` / `ROLLBACK`).
- Routing table for single-host deployments.
- Structured Neo4j error codes.
- Graceful shutdown.

See [docs/bolt.md](../docs/bolt.md) for the operator guide.

### TCK Conformance â€” 90.7 % at Parser Level

The [openCypher Technology Compatibility Kit (TCK)][tck] is now integrated
as a CI gate. At the parser level, GoGraph passes **90.7 %** of the 3 897
TCK scenarios (3 534 run, 100 % pass on run; 363 skipped for grammar gaps
documented in [DIVERGENCES.md](../docs/tck/DIVERGENCES.md)).

A dedicated `.github/workflows/tck.yml` workflow fails the build if the rate
falls below 90 %.

**Execution-level TCK** (end-to-end query results verified against expected
output) passes **10.4 %** of scenarios at this tag. The remaining gaps are
catalogued in `docs/tck/DIVERGENCES.md` and are being addressed in subsequent
release candidates.

### Soak Testing

The Bolt server passes a 60-second mixed-workload soak test (32 goroutines,
auto-committed and explicit-transaction queries) with:
- Zero goroutine leaks (`goleak`).
- Zero race conditions (`-race`).
- Heap delta within expected bounds.

A 4-hour / 1 024-connection variant is available via `SOAK_FULL=1`. The CI
soak report is committed to `soak-artefacts/bolt-soak-ci-report.md`.

## Upgrade Guide

**From v1.x:** No breaking changes to the `graph/`, `search/`, `store/`, or
`bench/` packages. The new `cypher/` and `bolt/` packages are additive.

The only change to an existing API is in `cypher/exec.ResultSet.Next`: the
`Record` map is now reused across calls. This was already the documented
contract ("callers must copy values they need to retain beyond the next
`Next` call"), so correct callers are unaffected.
