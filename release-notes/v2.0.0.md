# Release Notes — GoGraph v2.0.0

**Date:** 2026-05-31
**Tag:** `v2.0.0`
**Module:** `gograph` (`github.com/xumiga/gograph`)
**Status:** Stable

---

## Overview

GoGraph v2.0.0 is the first major release since the 1.x line. It promotes the
library from a graph-algorithm toolkit into a **query-capable, server-ready
graph engine**, and it ships the two compliance guarantees that define the
project:

- **100 % openCypher TCK-compliant at the execution level** — every one of the
  **3 897 scenarios** (16 006 steps) in the openCypher Technology Compatibility
  Kit passes end-to-end, with no `failed`, `undefined`, or `pending` steps. The
  regression gate (`tckExecutionBaseline = 3897` in `cypher/tck/runner_test.go`)
  rejects any pull request that lowers the count.
- **100 % ACID-compliant** — Atomicity, Consistency, Isolation, and Durability
  hold across the in-memory engine and every persistence backend, verified by
  the deterministic crash-injection battery and the WAL recovery tests.

This release supersedes and consolidates the two pre-release candidates that
led here:

- [v2.0.0-rc1](v2.0.0-rc1.md) — sprints 21–32: the Cypher execution engine, the
  Bolt v5 server, and the TCK harness at 90.7 % parser / 10.4 % execution.
- [v2.0.0-rc2](v2.0.0-rc2.md) — sprints 37–40: WAL durability bridging, 99.5 %
  parser, 25.8 % execution.

Everything from those candidates, plus the post-rc2 work described below, is
included in v2.0.0. The headline of the post-rc2 effort is driving
execution-level openCypher conformance from **25.8 % to 100 %**.

For the full, per-sprint change log see [CHANGELOG.md](../CHANGELOG.md). To
upgrade an existing 1.x deployment, follow the
[1.x → 2.x migration guide](../docs/migration-1-to-2.md).

## Highlights

### Cypher query engine — 100 % execution TCK conformance

GoGraph ships a complete Cypher query engine built on the Volcano/iterator
model. It covers reading (`MATCH`, `OPTIONAL MATCH`, `WHERE`, `RETURN`, `WITH`,
`ORDER BY`, `SKIP`, `LIMIT`, `DISTINCT`, `UNION`, variable-length paths,
`UNWIND`), writing (`CREATE`, `MERGE`, `SET`, `REMOVE`, `DELETE`,
`DETACH DELETE`, `FOREACH`), aggregation (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`,
`COLLECT`, `PERCENTILE_CONT`, `PERCENTILE_DISC`, `ST_DEV`), schema DDL
(`CREATE`/`DROP INDEX`, `CREATE`/`DROP CONSTRAINT`, `SHOW INDEXES`,
`SHOW CONSTRAINTS`), built-in procedures (`db.labels`, `db.indexes`,
`db.constraints`, and more), parameters, an LRU plan cache, and `EXPLAIN` /
`PROFILE` with `db_hits` accounting.

Where rc2 stood at 25.8 % execution conformance, v2.0.0 is at **100 %**. The
gap was closed across an extended audit-and-uplift programme that wired the
remaining operators, fixed result-equality semantics, and reconciled
behaviour against the openCypher 9 specification and the upstream reference
implementation — never from memory. The authoritative pass-rate table and the
historical progression live in
[`docs/tck/DIVERGENCES.md`](../docs/tck/DIVERGENCES.md) and
`cypher/tck/conformance_history.go`. The full language reference is in
[`docs/cypher.md`](../docs/cypher.md).

### Bolt v5 server

GoGraph can act as a drop-in, Neo4j-compatible graph server, accepting
connections from `neo4j-go-driver` v5 and `cypher-shell`. It speaks Bolt
protocol versions 5.0–5.6 (with a 4.4 fallback), supports basic authentication
and TLS, explicit transactions (`BEGIN` / `COMMIT` / `ROLLBACK`), a routing
table, structured Neo4j error codes, and graceful shutdown. Internal error
detail is now sanitised before reaching the client and logged server-side with
the session correlation ID. See [`docs/bolt.md`](../docs/bolt.md) for the
operator guide.

### ACID hardening (F1–F5)

The transactional core was audited for five concrete gaps and all five were
closed:

- **Atomicity** — transactions commit through transaction-grouped WAL records
  so a write batch becomes visible all-or-nothing.
- **Consistency / checkpoint self-sufficiency** — a checkpoint includes the
  typed mapper before truncating the WAL, so a snapshot is always
  self-sufficient on recovery.
- **Isolation** — an isolation barrier (`lpg.Graph.ApplyAtomically` /
  `Graph.View`) guarantees no reader ever observes a partial transaction,
  across both `store/txn` and the Cypher engine.
- **Durability** — the WAL fsyncs its parent directory on file creation, and a
  durable-but-unapplied transaction is reconciled on recovery via the
  `ErrCommittedNotApplied` sentinel, proven by a cross-process recovery test.

These properties are preserved for the in-memory engine and for every
persistence backend.

### Production readiness

The Bolt server and Cypher engine gained the bounded-resource and
observability surface required to run in production: a self-contained
Prometheus text-exposition backend (`internal/metrics/prometheus`), latency
histograms and paired error counters on every public blocking API, a
server-side statement-timeout cap, an insecure-default startup warning, and a
per-query statement-`now` registry that removes a process-global race on
`NOW()` / `date()` / `datetime()` under concurrency.

### Documentation and examples

The godoc effort added **+95 runnable `Example*` functions** (from 2 to 97)
across all ~44 exported packages, package-level doc comments where they were
missing, and an entrypoint map in the root `doc.go`. The examples-excellence
pass brought all **25** runnable examples to faithfulness parity, each with a
`run(io.Writer)` entrypoint, a test, and a README, indexed by
`examples/README.md`.

## What changed since rc2 (grouped)

This section explains the why and the what; the per-sprint detail is in
[CHANGELOG.md](../CHANGELOG.md).

### openCypher execution conformance: 25.8 % → 100 %

The single largest body of work. Beyond wiring operators, the uplift required
correcting result-equality and ordering semantics, temporal-value handling,
pattern-comprehension and variable-length-expand edge cases, multigraph
adjacency semantics, and named-path reconstruction — each change validated
against the relevant TCK feature file rather than assumed. The execution gate
is now pinned at the full scenario count and enforced on every PR.

### Reliability and concurrency

A concurrent read-plus-write panic in the Cypher engine was eliminated by
building the plan inside `Graph.View` / `ApplyAtomically` against a
`nowAwareRegistry`, with a dedicated race regression test. The full suite is
race-clean.

### Performance

The Cypher read path shed allocations across several committed wins (for
example, the IC1 full-scan benchmark improved while execution conformance was
held at 3 897). Headline figures, methodology, and cross-concurrency results
are in [`docs/benchmarks/cypher.md`](../docs/benchmarks/cypher.md).

### Continuous integration

The repository is mirrored to GitHub with a full Actions CI pipeline. All jobs
are green: build, test, race, lint (`golangci-lint` v2.12.2), vet, the
dedicated `TestTCKExecution` gate (with `-timeout 600s`), coverage gate, and
`govulncheck`.

## Breaking changes

v2.0.0 keeps the breaking surface intentionally narrow. Most 1.x code that
used only the in-memory graph and the search algorithms compiles unchanged. The
breaking changes are:

1. **The v1 WAL record format is removed.** The legacy untagged WAL format, the
   `store/txn.NewStore` constructor that produced it, and the
   `recovery.OpenString`, `recovery.OpenWithCodec`, and
   `recovery.OpenWithOptions` read wrappers (and their `*Ctx` variants) **no
   longer exist**. There is **no in-place v1 WAL reader**: a v1 WAL corpus is
   rejected on open with `recovery.ErrUnsupportedRecordVersion`. A v1 corpus
   must be migrated to a typed v2 store with a **1.x build before upgrading**.
   On-disk **snapshot** directories are unaffected — a v1 snapshot still loads.
2. **`graph.Graph.AddNode` and `AddEdge` now return `error`.** With the default
   (unbounded) shard capacity they never fail; with
   `adjlist.Config.MaxShardCapacity` set they return `adjlist.ErrShardFull`.
3. **`centrality.WeightedBetweenness` signature change** — from
   `([]float64)` to `([]float64, error)`, for consistency with
   `centrality.PageRank` and `centrality.PersonalisedPushPageRank`; it also
   rejects NaN/±Inf weights via `search.ErrInvalidInput` and strictly negative
   weights via `search.ErrNegativeWeight`.
4. **New bounded-resource defaults** are applied to several previously
   unbounded surfaces (`bolt/server` connection / message / in-flight caps,
   `cypher.EngineOptions.PlanCacheCapacity`). Raising a cap is explicit, via the
   relevant `Options` struct.
5. **Toolchain pin** — the module requires Go 1.26 with `toolchain go1.26.3`.

Each breaking change carries its full rationale in the corresponding
[CHANGELOG.md](../CHANGELOG.md) entry and in the migration guide.

## Migrating from 1.x to 2.0

The authoritative, step-by-step walkthrough — with copy-pasteable diffs — is in
[`docs/migration-1-to-2.md`](../docs/migration-1-to-2.md). In short:

0. **Migrate a v1 WAL corpus first (only if you have one).** Replay the v1 WAL
   on a 1.x build, re-commit through `txn.NewStoreWithCodec` /
   `NewStoreWithOptions` against a fresh WAL, then upgrade. A store that has
   taken at least one self-sufficient checkpoint and truncated its v1 WAL prefix
   needs no manual step.
1. **Bump the module** to `go 1.26` + `toolchain go1.26.3`; run
   `go mod tidy && go mod verify`.
2. **Handle the new error returns** on `AddNode` / `AddEdge`.
3. **Adopt the Cypher / Bolt surface** (optional; entirely new in 2.x).
   Remember that `cypher.Result.Close()` is mandatory.
4. **Migrate the persistence layer** if used: replace `txn.NewStore` with
   `NewStoreWithCodec` / `NewStoreWithOptions`, and replace the removed
   `recovery.Open*` wrappers with `recovery.Open` / `OpenCtx` plus explicit
   `recovery.Options` codecs.
5. **Adopt the new bounded-resource defaults**, raising caps explicitly where
   you legitimately need to.

## Soak evidence and provenance (honest disclosure)

Release gate #4 in [`docs/semver.md`](../docs/semver.md) asks for a full
1 024-connection, 4-hour soak against the release commit. The honest status at
the `v2.0.0` tag is:

- **Bolt 4-hour soak: PASS — but at commit `b5453b9`, not at the tag.** The
  `v2.0.0` tag is six commits ahead of `b5453b9` (`#1213`–`#1216`: CI-action
  bumps, a `golangci-lint` fix, a coverage-gate fix, and two flaky-test
  fixes). None of those six commits touches a runtime hot path; they are
  infrastructure and test-only changes.
- **Cypher full 4-hour soak: NOT yet recorded.** Only a 30-minute Cypher
  read/write soak (2026-05-21) is archived. It passed with zero goroutine
  leaks, zero races, and bounded heap.
- **Soak-smoke against the release commit: PASS.** A 60-second run of the soak
  harness (`make soak-smoke`, 8 readers, 15-second sampling) against the tagged
  commit completed cleanly: ~327 000 reads, ~1 200 writes, 29 snapshot rebuilds,
  goroutine count stable at 11 with no growth, and bounded heap across five
  snapshots.

**Decision for v2.0.0:** the release ships today on the strength of the
soak-smoke against the tag plus the 4-hour Bolt soak at `b5453b9`, with the
full 4-hour soak being re-run against the tag out-of-band. This disclosure is
deliberate — no soak result is overstated. The CI soak report is committed at
`soak-artefacts/bolt-soak-ci-report.md`; the canonical execution path is the
weekly `Soak` GitHub Actions workflow.
