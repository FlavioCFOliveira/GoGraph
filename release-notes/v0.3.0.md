# Release Notes ‚Äî GoGraph v0.3.0

**Date:** 2026-06-13
**Tag:** `v0.3.0`
**Module:** `github.com/FlavioCFOliveira/GoGraph`
**Status:** Pre-1.0 (unstable public API)

## Overview

GoGraph v0.3.0 is the third published release of GoGraph, a Go module
for graph **persistence**, **manipulation**, and ‚Äî above all ‚Äî **fast
search**, designed to scale from in-memory graphs to graphs that exceed
RAM.

This is a pre-1.0 **MINOR** release. It is **additive** over `v0.2.0`,
and its headline is a deep **reliability and robustness hardening pass**
that consolidates the findings of three successive code audits, together
with four additive features. No exported identifier was removed or
renamed, and only one consumer-visible default was tightened (the
CSV/GraphML import byte ceiling).

The hardening reaches across every layer of the module:

- the **Cypher engine** ‚Äî correctness and robustness fixes spanning
  `reduce()`, openCypher equivalence semantics for `DISTINCT` and
  grouping, parser-panic and arithmetic-overflow guards,
  `ParameterMissing`, type-error fidelity, and `SET`-clause spec
  fidelity;
- the **engine, Bolt protocol, and search** layers ‚Äî the autocommit
  read path no longer takes the writer lock, `HopcroftKarp` validates
  its input, Cypher temporal values are encoded on the wire as the
  PackStream temporal Structs real drivers expect, and client-fault
  conditions are classified correctly;
- the **import/export and observability** surface ‚Äî CSV/GraphML
  token-OOM ceilings, fail-stop on XML-illegal characters, Prometheus
  name sanitisation, portable non-finite-number encoding, CSV
  byte-order-mark stripping, and a JSON Lines line cap;
- the **durability path** ‚Äî a recovery-promotion parent-directory
  `fsync` and a checkpoint constraint-survival fail-safe; and
- the **test-battery and release-gate infrastructure** ‚Äî an additive
  TCK error-type fidelity gate, a whole-repository concurrency-doc
  scan, and the invariant checkers run against real shape-generator
  output.

Under [Semantic Versioning](https://semver.org/), a `0.y.z` version
signals that the public Go API is **not yet stable**: it may change
without a major-version bump while the module matures toward `1.0.0`.
Pin the exact version you depend on. This release introduces no breaking
change; read the [Upgrade notes](#upgrade-notes) for the single tightened
default before upgrading.

Both compliance guarantees that define the project continue to hold
without regression at this version:

- **100 % openCypher TCK-compliant at the execution level** ‚Äî every
  scenario in `cypher/tck/features/` passes, with no `failed`,
  `undefined`, or `pending` step. The execution gate stands at
  **3 897 / 3 897 scenarios** (16 006 / 16 006 steps). The regression
  gate (`tckExecutionBaseline = 3897` in `cypher/tck/runner_test.go`)
  rejects any change that lowers the count.
- **100 % ACID-compliant** ‚Äî Atomicity, Consistency, Isolation, and
  Durability hold across the in-memory engine and every persistence
  backend. Durability is verified by the WAL recovery tests in
  `store/wal` and `store/recovery` and by the deterministic
  crash-injection battery in `internal/crashinject`.

Install with:

```bash
go get github.com/FlavioCFOliveira/GoGraph@v0.3.0
```

The Go module path is `github.com/FlavioCFOliveira/GoGraph` with **no**
`/vN` suffix, which is Semantic-Import-Versioning-correct for a `0.x`
line.

## Highlights

- **`reduce()` list folding** is now reachable from the public API:
  `reduce(acc = init, x IN list | expr)` parses, type-checks, and
  evaluates end to end.
- **Correct PackStream temporal wire encoding** over Bolt ‚Äî Cypher
  dates, times, datetimes, and durations now reach a real driver as the
  temporal Structs it can hydrate into `neo4j.Date` / `neo4j.Duration`
  / `time.Time`, instead of plain strings.
- **A public `metrics` facade package**, so the documented Prometheus
  wire-up compiles from outside the module.
- **A faster adjacency list under hubs** ‚Äî `AddEdge` is amortised O(1)
  and `RemoveAllEdgesFrom` is O(d) for high-degree nodes, eliminating
  the previous quadratic behaviour.
- **Cypher robustness** ‚Äî the parser no longer crashes the process on
  malformed input, deep operator chains can no longer overflow the
  stack, integer and floating-point overflow are reported as typed
  errors, and a missing query parameter raises `ParameterMissing`.
- **openCypher equivalence semantics** for `DISTINCT` and grouping, so
  `NaN ‚â° NaN` and nested nulls are grouped as the specification
  requires.
- **`SET`-clause fidelity** ‚Äî four silent divergences (including a
  latent node-mislabel hazard) now fail loud and correct.
- **A ghost-node durability fix** ‚Äî a node deleted through Cypher stays
  deleted across a store reopen.
- **Bounded, fail-stop import/export** ‚Äî CSV/GraphML token-OOM
  ceilings, fail-stop on XML-illegal characters, a JSON Lines line cap,
  CSV byte-order-mark stripping, and portable non-finite-number
  encoding.
- **Hardened observability** ‚Äî Prometheus metric-name sanitisation
  closes an exposition-injection vector through the public facade.
- **Tighter durability ordering** ‚Äî a recovery-promotion
  parent-directory `fsync`, a checkpoint constraint-survival fail-safe,
  and an exclusive multi-process WAL lock.

## What is new

### `reduce()` list-folding expression (`cypher/`)

The `reduce(acc = init, x IN list | expr)` accumulator expression is now
wired through the full Cypher pipeline: a grammar production and a new
`*ast.ReduceExpr` AST node, semantic analysis and schema checking, the
IR (aggregation and pattern-comprehension paths), and the evaluator.
Folding a list into a single accumulated value is therefore reachable
from the public API. A null source list yields null, matching the
openCypher specification. The feature is additive and does not conflict
with any TCK-covered semantics (#1426).

### PackStream temporal wire encoding (`bolt/`)

The Bolt value encoder previously had no case for the six Cypher
temporal value kinds, so `Date`, `LocalTime`, `Time`, `LocalDateTime`,
`DateTime`, and `Duration` fell through to a string representation and
reached the client as plain PackStream strings ‚Äî breaking `.AsTime()`
and temporal arithmetic in a real neo4j driver. Each kind is now encoded
as the canonical PackStream temporal Struct the neo4j-go-driver hydrator
expects. `DateTime` is **Bolt-version- and time-zone-aware**: Bolt v5.0+
uses the UTC convention (a true UTC epoch second with an offset or a
named-zone Struct), while v4.4 uses the legacy wall-clock convention.
The negotiated Bolt version is threaded into the session so the correct
convention is chosen per connection (#1434).

### Public `metrics` facade package (`metrics/`)

The metrics types were previously confined to internal packages, which
made the observability wire-up documented in `docs/metrics.md`
uncompilable for an external consumer. A new top-level `metrics` package
now re-exports them via type aliases ‚Äî `metrics.Backend`,
`metrics.Registry`, `metrics.SetBackend`, and
`metrics.NewPrometheusRegistry` ‚Äî so the documented `/metrics` endpoint
wire-up compiles outside the module. The aliases keep a single
implementation: a `*prometheus.Registry` satisfies both the internal and
the public `Backend` interface without conversion.

### Filesystem fsync-failure fault modes (`internal/testfs`)

The internal fault-injection library gains two fsync-failure fault
modes, `Faults.FailSyncAfter` (the first N syncs succeed, then every
sync fails) and `Faults.ReturnEIOOnSync` (every sync fails). Both return
the new `ErrSyncFailed` sentinel and discard the unsynced suffix, so
recovery-based durability tests can assert that an unacknowledged commit
leaves no trace without standing up the full crash-injection harness.
This is test-only infrastructure with no production surface.

### Additive TCK error-type fidelity gate (`cypher/tck/`)

A new conformance gate asserts that the engine raises the correct
openCypher error *type* ‚Äî not merely that a negative scenario fails ‚Äî
built on the engine's existing semantic-error vocabulary. The gate is
purely additive: the execution-level result count is unchanged at
3 897 / 3 897, and the two error categories whose mapping is genuinely
ambiguous are deliberately excluded rather than asserted incorrectly
(#1443).

## What changed (behavioural)

This release introduces **no breaking API change**. It tightens one
consumer-visible default and corrects one consumer-visible behaviour:

- **The Cypher `=~` regex operator now matches correctly.** It was
  silently parsed as `=` (string equality), so `'abc' =~ '[a-z]+'`
  returned false. It now performs an anchored full-match regular
  expression (`\A(?:‚Ä¶)\z`, openCypher / Java `String.matches`
  semantics). Any query that relied on the old (incorrect) behaviour ‚Äî
  in particular `=~` used as an authorisation or allow/deny predicate ‚Äî
  will now evaluate differently; review such queries (#1479).
- **CSV and GraphML single-token import RAM is bounded** by lowering the
  default import byte ceiling for those two decoders to **128 MiB**
  (from 1 GiB), capping the peak memory a single oversized token can pin
  while a record is assembled. The ceiling remains configurable, and
  `MaxBytes <= 0` still means unlimited (#1436).

Two further changes are surface clarifications rather than behavioural
shifts for existing callers: `reduce(...)` now parses and executes where
it previously failed at parse time (#1426), and the `cypher/ir/rewrite`
package now documents its experimental status and is guarded by an
import-graph test.

## Reliability and robustness hardening

The bulk of v0.3.0 is correctness work consolidating three successive
audits. It is grouped here by area; every item is faithful to a
committed fix.

### Cypher correctness and robustness

- **openCypher equivalence for `DISTINCT` and grouping.** Grouping and
  de-duplication now compare by openCypher *equivalence* rather than
  equality, so `NaN ‚â° NaN` and nested nulls group correctly
  (CIP2016-06-14) (#1420).
- **No process crash on malformed input.** The parser wraps `p.Script()`
  in `recover()` for malformed `WITH`-clause and pipe-in-argument
  queries (#1422), and the pre-parse guard is extended to `CASE` nesting
  and binary-operator chains. A deep operator chain can no longer
  overflow the stack during semantic analysis, because `checkExpr`
  carries a recursion-depth budget (#1424).
- **Missing parameters are typed errors.** Referencing a query parameter
  that was not supplied raises `ParameterMissing` instead of evaluating
  to a silent null (#1431), and numeric-literal property access such as
  `(5).foo` raises `InvalidArgumentType` (#1430).
- **Arithmetic overflow is detected, not wrapped.** Typed overflow
  errors are raised for `sum()` accumulation (#1427), `MinInt64 / -1`
  integer division (#1419), `toInteger()` boundary rounding at `2^63`
  (#1428), pure integer literals over 19 digits, and exponent-form float
  literals that overflow `float64` (#1421). The `Parse` and `ParseStrict`
  entry points now share one normaliser pipeline, removing a divergence
  on valid float literals (#1423).
- **Result caps apply to write-only queries.** `MaxResultRows` is now
  enforced on write-only queries, returning `ErrResultRowsExceeded` and
  rolling the write back atomically (#1425). A leading `WITH ... WHERE`
  is seeded with a single-row argument so it no longer crashes.

### `SET`-clause spec fidelity

An exhaustive audit of the `SET` clause surfaced four silent-divergence
defects; none crashed, but each diverged from openCypher behaviour and
one was a latent data-integrity hazard. All four now fail loud and
correct (#1455, #1456, #1457, #1458):

- Setting or removing a **label on a relationship** is rejected at
  semantic analysis with a `TypeError`. This also closes a latent hazard
  that reinterpreted an edge-position counter as a node id and could
  silently mislabel the wrong node.
- Whole-entity RHS typing: `SET n = null` clears all properties (the same
  as `SET n = {}`), `SET n += null` is a no-op, and a non-null non-map
  literal such as `SET n = 5` is a compile-time `TypeError`.
- A nested map property raises `InvalidPropertyType`.
- A non-map parameter to `SET n = $p` raises a `TypeError`.

### Transactions, isolation, and atomicity

- An explicit transaction holds the visibility lock for its entire
  lifetime, so concurrent readers block while a transaction is open ‚Äî
  closing a read-uncommitted isolation gap (#1412).
- `ExplicitTx.Commit` is rejected after a failed `Exec`, preventing a
  partial transaction from becoming durable (#1413), and surfaces
  `ErrUndoFailed` when undo replay fails on a WAL `fsync` error.
- DDL index and constraint registration and backfill now run inside the
  visibility barrier (#1417).

### Engine, protocol, and search robustness

- **Bolt autocommit reads no longer take the writer lock.** A read-only
  autocommit statement is routed through `RunAny` rather than
  `RunInTxAny`, so it does not acquire the single-writer lock (#1432).
- **`HopcroftKarp` validates its input.** Maximum-bipartite-matching
  validates `nLeft` and returns `ErrInvalidInput` instead of panicking
  on malformed input (#1433).
- **Client faults are classified as client errors.** Bolt and Cypher now
  distinguish client-fault conditions from genuine server errors (#1435).

### Import, export, and observability

- The **GraphML writer fails stop on XML-illegal characters** in string
  properties instead of silently emitting `U+FFFD`, on both write paths,
  and validates node ids on the plain `Write` path too (#1437).
- The **JSON Lines reader** returns a typed `ErrLineTooLong` for an
  oversized single line (#1442).
- The **CSV reader strips a leading UTF-8 byte-order mark** so
  spreadsheet exports parse correctly (#1441).
- **GraphML emits portable `xs:double`** `NaN` / `INF` / `-INF`, and the
  JSON Lines non-finite contract is documented (#1440).
- The **DOT writer emits bare node statements** so isolated vertices are
  no longer dropped from the output (#1439).
- The **Prometheus exposition writer sanitises metric names** to the
  valid grammar, closing an exposition-injection vector through the new
  public metrics facade (#1438); `WriteText` now returns its accumulated
  write error.

### Durability and recovery

- **Crash-safe snapshot-backup promotion.** Recovery `fsync`s the parent
  directory after promoting a snapshot backup, so a crash between the
  promotion rename and the directory flush cannot lose the promoted
  snapshot (A1-F4, #1454). This mirrors the publish-path `fsync` ordering
  already present elsewhere in the persistence stack.
- **Constraints survive a checkpoint unconditionally.** A checkpoint
  gates WAL truncation on `Graph.HasConstraints`, so declared
  constraints survive a checkpoint and WAL truncation even when the
  embedder has not wired `checkpoint.WithConstraintSpecs` (#1464).
- **Multi-process WAL protection.** The WAL acquires an exclusive OS file
  lock on `Open` to prevent multi-process WAL corruption (#1416),
  `fsync`s after `ftruncate` in `poison()` so truncation is durable
  across a host crash (#1414), and the snapshot writer `fsync`s the
  `indexes` subdirectory after writing index files for directory-entry
  durability (#1415).

### Cypher DELETE durability

A node deleted through Cypher previously failed to emit an
`OpRemoveNode` WAL frame from the mutator adapter, so it resurrected as a
ghost on the next store reopen because WAL replay had no tombstone to
apply. The adapter's `RemoveNode` now emits the frame, matching every
other WAL mutator sibling, so a Cypher-deleted node stays deleted across
a reopen (#1411).

## Security-relevant hardening

This release ships no toolchain change ‚Äî the Go toolchain remains
**go1.26.4**, the baseline established in `v0.2.0`, and `govulncheck
./...` is clean. Several robustness fixes nonetheless harden the
module's exposed surfaces:

- **Prometheus exposition-injection** through the public metrics facade
  is closed by metric-name sanitisation (#1438).
- **Multi-process WAL corruption** is prevented by an exclusive OS file
  lock acquired on `Open` (#1416).
- **Untrusted-import memory amplification** is bounded by the 128 MiB
  CSV/GraphML single-token ceiling (#1436), the JSON Lines line cap
  (#1442), and fail-stop GraphML XML validation (#1437).

A dedicated, phased security audit (**SEC-2026-06-14**) was then run
across six domains and its **13** findings fixed (tasks #1467‚Äì#1479); the
full report is
[docs/security-audit-2026-06-14.md](../docs/security-audit-2026-06-14.md).
The remediation closes, all reachable from untrusted input:

- **Memory-exhaustion DoS** ‚Äî snapshot decoders clamp eager allocation
  before the CRC/size gate (a hostile snapshot could OOM `recovery.Open`)
  and the Cypher evaluator bounds per-evaluation list growth, so a tiny
  `reduce()`/comprehension query can no longer exhaust host memory
  (#1467, #1468, #1469, #1475).
- **Uncancellable queries** ‚Äî `reduce()`/comprehension/quantifier loops
  now honour `context` cancellation (#1477); variable-length paths gained
  a per-query edge budget and a default hop ceiling (#1478).
- **Bolt credentialed auth** ‚Äî `BasicAuthHandler` authenticates at
  `LOGON` for Bolt ‚â• 5.1, so credentialed auth works with modern drivers
  instead of forcing `NoAuth` (#1470).
- **`=~` regex correctness** ‚Äî fixed and anchored per openCypher, closing
  a fail-open hazard in authorisation predicates (#1479; see Upgrade
  notes).
- **Untrusted-graph analytics** ‚Äî `TransitiveClosure`, WCC and Kruskal
  compact over the live node set and `UnionFindSlice` uses 64-bit
  indices, defusing a hash-flood amplification (#1474, #1476).
- **Export and supply chain** ‚Äî opt-in CSV formula-injection sanitisation
  (#1471), and every release-pipeline GitHub Action pinned to a commit
  SHA with the SBOM generator pinned to an exact version (#1472, #1473).

Both compliance invariants held throughout (openCypher TCK 3 897 / 3 897,
ACID), and `go test -race`, golangci-lint, staticcheck and `govulncheck`
are clean.

## Performance

The one performance change in this release improves **adjacency-list hub
operations** (`graph/adjlist`). `AddEdge` is now amortised **O(1)** and
`RemoveAllEdgesFrom` is **O(d)** for high-degree (hub) nodes, replacing
the previous quadratic behaviour. The append fast path reuses the same
backing array in place when spare capacity exists and publishes new
slice headers atomically (preserving the lock-free read contract), and
falls back to a geometric (√ó2, minimum 4) grow-and-copy only O(log d)
times for a degree-d hub; bulk removal acquires the source shard lock
once and publishes the cleared neighbour list atomically. A
degree-10 000 hub is now roughly 11.6√ó slower than a degree-1 000 hub,
where it was previously 50‚Äì100√ó slower.

The headline single-threaded benchmark figures are otherwise unchanged
from [v0.2.0](v0.2.0.md#performance); re-measure on your own hardware
before relying on any absolute number. The per-release benchmark and
load-test report, including the 1 / 8 / 64 / 256 / 1 024-goroutine
concurrency sweep, is published in
[docs/benchmarks/v0.3.0.md](../docs/benchmarks/v0.3.0.md).

## Upgrade notes

This release is **API-additive** over `v0.2.0`: no exported identifier
was removed or renamed, and there is **no breaking API change**. One
consumer-visible default changed and one consumer-visible behaviour was
corrected; both should be reviewed before upgrading.

1. **The Cypher `=~` regex operator now matches correctly.** Previously
   it was silently evaluated as `=` (string equality) ‚Äî `'abc' =~
   '[a-z]+'` returned false ‚Äî so the regex path was effectively dead. It
   now performs an anchored full-match regular expression (`\A(?:‚Ä¶)\z`,
   openCypher / Java `String.matches` semantics). **Action:** review any
   query that used `=~`, especially as an authorisation or allow/deny
   predicate, because results will change (a substring that used to
   "match" via equality no longer does, and a real pattern now matches).
   (#1479)
2. **CSV and GraphML import byte ceiling lowered to 128 MiB.** The
   default `MaxBytes` for the CSV and GraphML decoders is now **128 MiB**
   (previously 1 GiB), to bound the peak memory a single oversized token
   can pin while a record is assembled. Input above the ceiling is
   rejected with the typed `ErrInputTooLarge`. If you legitimately import
   CSV or GraphML payloads larger than 128 MiB, raise `MaxBytes`
   explicitly (or set `MaxBytes <= 0` for unlimited). The JSON Lines
   decoder ceiling is unchanged.

Two further changes are clarifications rather than migrations: queries
that use `reduce(...)` now parse and execute where they were previously
rejected at parse time, and the `cypher/ir/rewrite` package is
explicitly documented as experimental and not part of the stable public
surface.

Because the public API is pre-1.0, continue to **pin the exact version**
you depend on.
