# Release Notes — GoGraph v0.6.0

**Date:** 2026-06-28
**Tag:** `v0.6.0`
**Module:** `github.com/FlavioCFOliveira/GoGraph`
**Status:** Pre-1.0 (unstable public API)

---

## Overview

GoGraph v0.6.0 is the eighth published release of GoGraph, a Go module
for graph **persistence**, **manipulation**, and — above all — **fast
search**, designed to scale from in-memory graphs to graphs that exceed
RAM.

This is a pre-1.0 **MINOR** release. It widens the **analytics and
Cypher surface** while landing a deep reliability and openCypher
conformance hardening pass. The headline additions are:

- **Four new centrality measures** in `search/centrality` — Closeness
  (Wasserman–Faust normalisation), Harmonic (Boldi–Vigna), Eigenvector,
  and Katz — each with a context-aware variant.
- **Node-label round-trip** in the JSONL and GraphML exporters, so a
  graph exported and re-imported preserves its label set.
- **Cypher map projection** — `RETURN n{.name, .*, k: expr, var}` — with
  selected-property, all-properties, literal-entry, and variable-selector
  forms.
- **`shortestPath()` / `allShortestPaths()` with per-instance
  relationship typing**, correct across parallel edges in a multigraph.
- A **columnar edge-property storage tier** and a
  **deterministic-simulation-testing (DST)** infrastructure expansion.

## Semantic-versioning rationale

The bump is **MINOR** under [Semantic Versioning](https://semver.org/).
Every change since `v0.5.0` is **additive** to the exported Go API — new
centrality functions, new Cypher capabilities, new options, new
`CSR.Validate` boundary check — or an **internal** fix that restores a
previously-documented contract. The 231 commits landed since `v0.5.0`
were surveyed (`git log v0.5.0..HEAD`) and **no breaking change to the
exported Go API** was found, so there is no "Breaking" note in this
release. As a `0.y.z` release the public API nonetheless remains
unstable: it may change without a major-version bump while the module
matures toward `1.0.0`. **Pin the exact version you depend on.**

## Compliance mandates — affirmed

Both compliance guarantees that define the project continue to hold
without regression at this version:

- **100 % openCypher TCK-compliant at the execution level** — every
  scenario in `cypher/tck/features/` passes, with no `failed`,
  `undefined`, or `pending` step. The execution gate stands at
  **3 897 / 3 897 scenarios** and **16 006 / 16 006 steps**. The
  regression gate (`tckExecutionBaseline = 3897` in
  `cypher/tck/runner_test.go`) is unchanged.
- **100 % ACID-compliant** — Atomicity, Consistency, Isolation, and
  Durability hold across the in-memory engine and every persistence
  backend, verified by the deterministic crash-injection battery
  (`internal/crashinject/`) and the WAL recovery tests
  (`store/wal/`, `store/recovery/`). This release strengthens that
  guarantee: WAL torn-tail detection, durable schema DDL across a
  WAL-truncating checkpoint, commit-time `NOT NULL` enforcement, and a
  rollback-correctness fix for no-op edge `CREATE` all close concrete
  durability and consistency gaps.

The Go toolchain remains **go1.26.4** (unchanged from `v0.5.0`), and
`govulncheck ./...` stays clean.

Install with:

```bash

```

The Go module path is `github.com/FlavioCFOliveira/GoGraph` with **no**
`/vN` suffix, which is Semantic-Import-Versioning-correct for a `0.x`

## Highlights

- **Four new centrality measures.** `Closeness`, `Harmonic`,
  `Eigenvector`, and `Katz` join the existing `Brandes` betweenness and
  `PageRank` analytics. Each ships a context-aware `…Ctx` variant;
  `Eigenvector` and `Katz` take an options struct and return the
  iteration count alongside the score vector. (#1800)
- **Interchange exporters carry node labels.** The JSONL and GraphML
  writers now preserve node labels across export and re-import. (#1793)
- **Cypher map projection.** `n{.name, .*, k: expr, var}` is now
  supported, composable in a single projection. (#1775)
- **Path functions with per-instance relationship typing.**
  `shortestPath()` and `allShortestPaths()` honour relationship-type
  predicates and per-instance relationship types, including across
  parallel edges. (#1685, #1690, #1691, #1692)
- **Per-instance by-handle edge properties.** `SET`/`REMOVE` on a bound
  relationship maintain per-instance edge properties addressed by
  handle, correct under parallel edges. (#1684, #1686)
- **`CSR.Validate` boundary check.** A new method and `ErrMalformedCSR`
  sentinel reject a malformed CSR snapshot before a search hot path can
  observe it. (#1762)
- **Columnar edge-property storage tier** cuts per-edge resident memory
  substantially on property-heavy graphs.
- **DST infrastructure.** A `SimDisk`-backed filesystem seam (`OpenFS`)
  backs snapshot, CSR file, and checkpoint paths, and the crash-storm
  scenario exercises full snapshot + WAL crash-recovery end to end.
  (#1546, #1740, #1752)

## What changed

### New centrality measures (`search/centrality`)

```go
func Closeness[W any](c *csr.CSR[W]) []float64
func ClosenessCtx[W any](ctx context.Context, c *csr.CSR[W]) ([]float64, error)

func Harmonic[W any](c *csr.CSR[W]) []float64
func HarmonicCtx[W any](ctx context.Context, c *csr.CSR[W]) ([]float64, error)

func Eigenvector[W any](c *csr.CSR[W], opts EigenvectorOptions) ([]float64, int, error)
func EigenvectorCtx[W any](ctx context.Context, c *csr.CSR[W], opts EigenvectorOptions) ([]float64, int, error)

func Katz[W any](c *csr.CSR[W], opts KatzOptions) ([]float64, int, error)
func KatzCtx[W any](ctx context.Context, c *csr.CSR[W], opts KatzOptions) ([]float64, int, error)
```

`Closeness` applies the Wasserman–Faust normalisation for disconnected
graphs; `Harmonic` implements the Boldi–Vigna harmonic-centrality
formulation, which is well-defined on disconnected graphs. `Eigenvector`
and `Katz` are iterative; they return the iteration count so callers can
observe convergence. Each carries an explicit concurrency contract in
its godoc.

### Cypher map projection

`RETURN n{.name, .*, k: expr, var}` is now supported, combining four
selector forms in a single projection: a property selector (`.name`),
the all-properties selector (`.*`), a literal entry (`k: expr`), and a
variable selector (`var`). (#1775)

### Path functions with per-instance relationship typing

`shortestPath()` and `allShortestPaths()` now honour relationship-type
predicates and per-instance relationship types, returning correct paths
across parallel edges in a multigraph. Directed `src == dst` shortest
cycles are found; the undirected case is safe; and context cancellation
is honoured during `allShortestPaths` enumeration. (#1685, #1690, #1691,
#1692, #1779, #1780, #1782)

### Interchange exporters carry node labels

The JSONL and GraphML writers carry node labels through export; GraphML
uses per-`(name, kind)` keys so heterogeneous node and edge property
keys round-trip without collision. (#1791, #1793)

### Reliability and openCypher conformance hardening

A series of audit rounds tightened correctness across the Cypher, Bolt,
storage, and search layers. Selected fixes:

- `ORDER BY … LIMIT 0` yields an empty result, not all rows. (#1801)
- Integer and Float order as one Number tier. (#1789)
- Nested aggregation is rejected at compile time. (#1804)
- A non-aliased compound grouping key reads its precomputed column. (#1803)
- The parser correctly discriminates binary minus in compact
  subtraction, and the variable-length bound rewrite is scoped to
  relationship brackets. (#1788, #1796, #1797, #1798)
- Relationship-uniqueness is enforced across comma-separated `MATCH`
  patterns. (#1777)
- WAL torn-tail detection prevents a corrupt length field from masking
  durable frames. (#1778)
- Schema DDL durably persists across a WAL-truncating checkpoint.
  (#1755, #1756)
- `NOT NULL` existence constraints are enforced at commit time. (#1754)
- A rolled-back no-op edge `CREATE` no longer deletes a pre-existing
  edge. (#1751)

### Performance

Numerous hot-path optimisations landed across the search, Cypher
execution, storage, and analytics layers — a columnar edge-property
tier, registry copy-on-write, scan arena reuse, an adaptive-parallelism
governor, snapshot write-path reductions, parallel index backfill, and
parallel variants of additional graph algorithms. The per-release
guard-band figures are recorded in
[docs/benchmarks/v0.6.0.md](../docs/benchmarks/v0.6.0.md); per-change
tracking lives in
[docs/benchmarks/history/](../docs/benchmarks/history/). Every
optimisation preserves the documented API contracts and both compliance
invariants.

## Upgrade notes

- **No migration required for engine users.** Stores written by earlier
  releases open without conversion. The new centrality functions, map
  projection, per-instance relationship typing, and node-label export
  are additive: existing code compiles and behaves unchanged.
- **New analytics are opt-in.** Call `Closeness`, `Harmonic`,
  `Eigenvector`, or `Katz` directly from `search/centrality` where you
  need them.
- **Pin the exact version.** As a `0.y.z` release the public API may
  still change without a major bump.
