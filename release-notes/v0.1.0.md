## Overview

GoGraph v0.1.0 is the **first published release** of GoGraph, a Go
module for graph **persistence**, **manipulation**, and — above all —
**fast search**, designed to scale from in-memory graphs to graphs that
exceed RAM.

This release **restarts the project's versioning at a pre-1.0
baseline**. Under [Semantic Versioning](https://semver.org/), a `0.y.z`
version signals that the public Go API is **not yet stable**: it may
change without a major-version bump while the module matures toward
`1.0.0`. Pin the exact version you depend on. The prior internal
numbering is intentionally abandoned; the complete development history
is preserved in git.

Despite the pre-1.0 API label, the two compliance guarantees that define
the project are already fully in force at this version:

- **100 % openCypher TCK-compliant at the execution level** — every
  scenario in `cypher/tck/features/` passes, with no `failed`,
  `undefined`, or `pending` step. The execution gate stands at
  **3 897 / 3 897 scenarios** (16 006 / 16 006 steps). The regression
  gate (`tckExecutionBaseline = 3897` in `cypher/tck/runner_test.go`)
  rejects any change that lowers the count.
- **100 % ACID-compliant** — Atomicity, Consistency, Isolation, and
  Durability hold across the in-memory engine and every persistence
  backend. Durability is verified by the WAL recovery tests in
  `store/wal` and `store/recovery` and by the deterministic
  crash-injection battery in `internal/crashinject`.

Install with:

```bash
go get github.com/FlavioCFOliveira/GoGraph@v0.1.0
```

The Go module path is `github.com/FlavioCFOliveira/GoGraph` with **no**
`/vN` suffix, which is Semantic-Import-Versioning-correct for a `0.x` /
`1.x` line.

For the full, per-change log see [CHANGELOG.md](../CHANGELOG.md).

## Highlights

- A unified, generic API surface across `graph/`, `graph/adjlist`, and
  `graph/csr` covering **directed and undirected graphs**,
  **multigraphs**, **self-loops**, and the **Labelled Property Graph**
  model with typed properties.
- A complete **openCypher engine** (`cypher/`): parser, planner, and
  execution engine with a plan cache, `EXPLAIN` / `PROFILE`, built-in
  functions and procedures, and WAL-durable writes — **100 %
  TCK-compliant at the execution level**.
- An **ACID persistence stack** (`store/`): a CRC32C-framed
  Write-Ahead Log, atomic snapshot directories, single-writer
  transactions, a background checkpointer, and snapshot + WAL-replay
  recovery — all crash-safe.
- A **Tier 2 mmap-backed CSR file format** (`store/csrfile`) with a
  validated header and zero-copy reinterpretation, plus semi-external
  BFS and PageRank that run without loading the graph into the Go heap.
- The canonical **analytical algorithm catalogue** (`search/`): BFS,
  DFS, Dijkstra, Bellman-Ford, A\*, bidirectional BFS, Yen k-shortest,
  topological sort, Tarjan SCC, biconnected components, Eulerian path,
  APSP, Brandes betweenness, PageRank, personalised PageRank, Leiden,
  label propagation, and a full network-flow suite (Dinic,
  Edmonds-Karp, push-relabel, Stoer-Wagner, min-cost max-flow).
- A **Bolt v5 server** (`bolt/`) compatible with `neo4j-go-driver` v5
  and `cypher-shell`, with TLS certificate hot-reload, graceful
  shutdown, and a fail-closed authentication default.
- **Interchange** with CSV, GraphML, Graphviz DOT, and JSON Lines.
- **25 runnable examples** and a suite of benchmark harnesses (LDBC SNB,
  DIMACS 9 SSSP, RMAT, soak, cross-library comparison).

## What is included

### Core graph (`graph/`)

- `graph` — the generic `Graph[N comparable, W]` contract and a sharded
  `Mapper` that interns external keys to dense internal `NodeID`s.
- `graph/adjlist` — a mutable, sharded, copy-on-write adjacency list
  with lock-free reads (writer side).
- `graph/csr` — an immutable Compressed Sparse Row snapshot for
  read-mostly analytics (reader side).
- `graph/generation` — refcount-protected atomic pointer swap for
  snapshot rotation across readers and a single writer.
- `graph/lpg` — the Labelled Property Graph model (vertex and edge
  labels, typed properties; `PropertyValue` covers `string`, `int64`,
  `float64`, `bool`, `time.Time`, `[]byte`), with stable per-edge
  handles and durable node tombstones.
- `graph/lpg/schema` — an optional declarative type schema with
  `Validate`.
- `graph/index`, `graph/index/label`, `graph/index/hash`,
  `graph/index/btree` — an index manager plus Roaring-bitmap label,
  sharded-hash exact-match, and B-tree range indexes.
- `graph/query` — a fluent `MATCH`-style pattern engine.
- `graph/io/csv`, `graph/io/graphml`, `graph/io/dot`, `graph/io/jsonl` —
  interchange formats.
- `ds` — a disjoint-set (union-find) primitive.

### Search and analytics (`search/`)

- `search` — BFS, iterative DFS, Dijkstra, Bellman-Ford, A\*,
  bidirectional BFS, Yen k-shortest, topological sort (Kahn), Tarjan
  SCC, biconnected components, Eulerian path, and APSP.
- `search/centrality` — Brandes betweenness, PageRank, personalised
  PageRank.
- `search/community` — Leiden, label propagation.
- `search/flow` — Dinic, Edmonds-Karp, push-relabel, Stoer-Wagner,
  min-cost max-flow.
- `search/extern` — semi-external BFS and PageRank over Tier 2 csrfile
  readers.

### Storage and persistence (`store/`)

- `store/wal` — a versioned, CRC32C-framed Write-Ahead Log.
- `store/snapshot` — atomic on-disk snapshot directories with a
  manifest and per-file CRC.
- `store/txn` — single-writer transactions (`Begin` / `Commit` /
  `Rollback`).
- `store/checkpoint` — a background WAL → snapshot folder goroutine.
- `store/recovery` — snapshot restore plus idempotent WAL replay on
  open.
- `store/csrfile` — the mmap-backed Tier 2 CSR file format, writer,
  reader, `Reinterpret` zero-copy helper, and fixture generator.
- `store/bulk` — a high-throughput bulk loader that bypasses the WAL.

### Cypher engine (`cypher/`)

- `cypher` — the openCypher-compatible parser, planner, and execution
  engine; WAL-durable writes via `NewEngineWithStore`.
- `cypher/parser`, `cypher/ast`, `cypher/sema`, `cypher/ir`,
  `cypher/plan`, `cypher/exec` — the parser-to-execution pipeline with
  a plan cache, `EXPLAIN` / `PROFILE`, and dbhits accounting.
- `cypher/funcs`, `cypher/procs` — built-in functions and procedures.
- `cypher/tck` — the openCypher TCK harness (execution 100 %, 3 897 /
  3 897 scenarios).

### Bolt server (`bolt/`)

- `bolt/proto`, `bolt/packstream` — Bolt v5 protocol and PackStream
  encoding (v5.0–v5.6 preferred, v4.4 fallback).
- `bolt/server` — a TCP server compatible with `neo4j-go-driver` v5 and
  `cypher-shell`, with TLS certificate hot-reload and graceful shutdown.

## Performance

Indicative benchmark numbers (Apple M4, 10-core, Go 1.26.3,
darwin/arm64):

| Operation | Throughput |
|---|---|
| `Mapper.Intern` (hot key) | 17 ns/op, 0 allocs |
| `adjlist.HasEdge` (hot cache) | 49 ns/op, 0 allocs |
| `csr.NeighboursByID` | 10.6 ns/op, 0 allocs |
| `csr.BuildFromAdjList` of 10^7 edges | ~51 ms |
| `search.BFS` on a 10^7-node chain | 38 ms, 1.25 MB peak, 0 allocs/call after warmup |
| `search.Dijkstra` on a 1M-node / 4M-edge random graph | ~320 ms |
| `search.BellmanFord` on 16K-vertex / 64K-edge | ~1.8 ms |

**Methodology.** Numbers are the median of five runs at `-count=5`,
captured with `benchstat`. Reproduce with
`make bench BENCH_PATTERN=. BENCH_COUNT=5` (see
[docs/profiling.md](../docs/profiling.md)). Hardware deltas should be
reported alongside any number that changes by more than the project's
5 % regression gate. The figures above are an indicative baseline for a
specific host; treat them as orders of magnitude rather than guarantees,
and re-measure on your own hardware.

## Installing and getting started

```bash
go get github.com/FlavioCFOliveira/GoGraph@v0.1.0
```

```go
package main

import (
	"fmt"

	"github.com/FlavioCFOliveira/GoGraph/graph/adjlist"
	"github.com/FlavioCFOliveira/GoGraph/graph/csr"
	"github.com/FlavioCFOliveira/GoGraph/search"
)

func main() {
	a := adjlist.New[string, int64](adjlist.Config{Directed: true})
	a.AddEdge("Lisbon", "Madrid", 624)
	a.AddEdge("Lisbon", "Paris", 1737)
	a.AddEdge("Madrid", "Paris", 1274)

	c := csr.BuildFromAdjList(a)
	src, _ := a.Mapper().Lookup("Lisbon")

	d, err := search.Dijkstra(c, src)
	if err != nil {
		panic(err)
	}
	for _, city := range []string{"Madrid", "Paris"} {
		id, _ := a.Mapper().Lookup(city)
		dist, _ := d.Distance(id)
		fmt.Printf("Lisbon -> %s : %d km\n", city, dist)
	}
}
```

The `examples/` directory contains 25 runnable demonstrations; see
[examples/README.md](../examples/README.md) for the categorised index.
Run any example with `go run ./examples/<NAME>/`.
