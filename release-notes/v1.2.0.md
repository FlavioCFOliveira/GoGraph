## GoGraph v1.2.0 — release-notes draft

Draft. Not yet tagged. The HEAD commit at the time of writing is
the closing commit of Sprint 21 (the audit closeout phase 2).

v1.2.0 is the first release cut from the post-v1.1.0 work that
spans seven sprints (14–20) of documentation accuracy, reliability
evidence, coverage uplift, observability wire-up, algorithmic
completeness, transactional generality, durable LPG persistence,
and the audit-driven correctness closeout. The release-notes
inventory below cross-links every closed rmp issue (#167–#187 from
the v1.1.0 audit follow-through and #188–#197 from the Sprint 21
audit closeout phase 2).

## Headline

- Generic durable LPG: `recovery.Open[N, W](dir, Options[N, W])`
  is the canonical typed-recovery entry point; supports any
  (string, int, int64, uint64, UUID, BinaryMarshaler) node-key
  type paired with (int64, float64, BinaryMarshaler) weights.
- Snapshot format v2: persisted labels (SLBL), persisted typed
  properties (SPRP), persisted secondary indexes (one file per
  index). v1 manifests still load.
- Real Johnson APSP via Bellman-Ford reweighting; mixed-sign
  weights supported, sparse-graph win of −61.75 % vs Floyd-Warshall.
- Observability wire-up complete across 56 files / 515 call-sites
  (search/, search/centrality/, search/community/, search/flow/,
  search/extern/, graph/io/, store/); headline benchmark geomean
  regression +0.68 % vs the pre-wire baseline.
- Audit correctness closeout: five algorithmic defects fixed
  (Eppstein rename, MinCostMaxFlow BF bootstrap, Diameter scratch
  aliasing, BCC forest articulation, NaN/Inf guards).
- SuiteSparse:GraphBLAS comparison column added to the
  cross-library benchmark.
- Library coverage 91.3 % aggregate (statement-weighted CI gate ≥ 85 %
  aggregate, ≥ 75 % per-package).
- errorlint enabled in the lint set; all 63 surfaced occurrences
  migrated to Go 1.20+ multi-wrap (`%w: %w`).

  `store/recovery` (closes #178).
- First publishable reliability soak — 30-minute mixed-workload

  heap delta −36.3 % (closes #181). The canonical 4-hour run is
  tracked as a follow-up.
- Observability claim corrected in `CHANGELOG.md` (closes #167).

### Coverage Uplift, GraphBLAS Baseline & Format Migration (Sprint 15)

- Rolling-upgrade compatibility harness — committed v1 fixtures
  under `testdata/v1/` and `format_compat_test.go` for `store/wal`,
  `store/snapshot`, `store/csrfile` (closes #175).
- Aggregate library coverage lifted from 79.1 % to 91.3 % with a

- SuiteSparse:GraphBLAS comparison column added to
  `docs/benchmarks/comparison.md` via `python-graphblas`, plus a

### Observability Completion & Algorithm Gap Closure (Sprint 16)

- `internal/metrics` wired across 56 non-test files / 515
  call-sites with `<package-path>.<ExportedSymbol>(.errors)`
  naming. Full inventory in `docs/metrics.md`. Geomean +0.68 %
  vs the pre-wire baseline (closes #168).
- `search.JohnsonAPSP` rewritten with the canonical Bellman-Ford
  reweighting via a virtual-source SPFA followed by per-source
  reweighted Dijkstra. Supports mixed-sign weights; surfaces
  `search.ErrNegativeCycle`. Sparse-graph win: −61.75 % vs
  Floyd-Warshall (closes #182).

### Transactional API Generalization (Sprint 17 — breaking)

- `store/txn` typed `Codec[N]` over the node-key type, with
  built-in codecs for `string`, `int`, `int32`, `int64`, `uint64`,
  UUID, and any `encoding.BinaryMarshaler`. WAL v2 frames carry
  the `OpRecordV2 = 0xFE` tag; v1 frames remain decodable. New
  constructor `NewStoreWithCodec` (closes #173).
- `store/txn.Tx.AddEdge` now records the per-edge weight via a
  `WeightCodec[W]`. New opcode `OpAddEdgeWeighted = 4`. New
  constructor `NewStoreWithOptions`; new sentinel
  `txn.ErrNoWeightCodec` (closes #174).

### LPG Snapshot Completeness (Sprint 18)

- LPG labels persisted in `labels.bin` (`SLBL` magic, packed
  string table, CRC32C trailer). Manifest version bumped to 2;
  v1 manifests still load (closes #170).
- LPG typed properties persisted in `properties.bin` (`SPRP`
  magic, fixed-width per-kind encoding covering all six
  `PropertyValue` kinds, 1 GiB cap) (closes #171).
- Secondary indexes persisted in `indexes/<name>.bin`, one file
  per registered index. Native Roaring serialisation for the
  label index; type-switched V codecs for hash and B+ tree.
  Snapshot/rebuild ratio 1.04× on a 10^5-node graph; CRC error
  or missing file is tolerated by falling back to a rebuild
  (closes #172).

### Generic Durable LPG (Sprint 19 — capstone)

- `store/recovery.Open[N, W](dir, Options[N, W])` and
  `OpenCtx[N, W]` are the canonical typed-recovery entry points.
  `recovery.Options` mirrors `txn.Options`;
  `Result.SnapshotSchemaVersion` is populated from the manifest.
  `OpenString`, `OpenWithCodec`, `OpenWithOptions` are
  Deprecated wrappers preserved for v1.x callers.
  `examples/21_typed_recovery` exercises (string, int64),
  (int64, int64), (int64, float64), and (UUID, float64)
  combinations (closes #169).

### Audit Correctness Closeout (Sprint 20)

The 2026-05-20 deep audit identified five algorithmic correctness
defects; all are now closed under `Specify → Implement → Test → Document`:

- `search.KShortestPathsLoopless` / `*Ctx` — honest name for what
  was previously called `EppsteinKShortest`. The shipped code is
  best-first enumeration over the loopless-path tree, NOT the
  heap-of-heaps construction of Eppstein 1998. The
  `EppsteinKShortest` symbols remain as `// Deprecated:` wrappers
  emitting both the old and the new metric names so dashboards
  keep working (closes #183).
- `flow.MinCostMaxFlow` — replaced a silent `if rc<0 { rc=0 }`
  clamp with a Bellman-Ford bootstrap that initialises potentials
  when the input network has negative-cost arcs. Negative-cycle
  reachable from src surfaces via the new sentinel
  `flow.ErrNegativeCycle`. The clamp is replaced by an internal
  invariant assert (closes #184).
- `search.Diameter` / `DiameterCtx` — separated the BFS scratch
  slices so the iFUB refinement's level filter no longer
  corrupts itself. Rapid-based property test against a brute
  V-BFS oracle (closes #185).
- `search.HopcroftTarjanBCC` — articulation root test now uses
  `p != start` (DFS-root identity) instead of `disc[p] != 0`
  (timer-derived). Roots of components 2+ in a forest are no
  longer mis-classified. Rapid property test against a
  remove-vertex brute oracle (closes #186).
- `search.BellmanFord` and `centrality.WeightedBetweenness` —
  NaN / +/-Inf edge weights now fail fast with
  `search.ErrInvalidInput`. Integer Weight types skip the
  validation pass (zero-value type-switch). BREAKING:
  `centrality.WeightedBetweenness` signature
  `([]float64) → ([]float64, error)` for consistency with
  `PageRank` and `PersonalisedPushPageRank`; also rejects
  negative weights via `search.ErrNegativeWeight` (closes #187).

### Audit Closeout Phase 2 (Sprint 21)

- `CHANGELOG.md` Unreleased section documents the Sprints 14-19
  feature work that previously lived only in git history (closes
  #188).
- README example count updated 20 → 21 plus a narrative entry for
  `examples/21_typed_recovery` (closes #189).
- `internal/metrics` package godoc no longer claims the wire-up
  is a "Sprint 11 or 12 follow-up" (closes #190).
- `search/centrality/goleak_test.go` — `BetweennessParallel`
  worker goroutines now covered by goleak (closes #191).
- `BetweennessParallelCtx` workers cascade cancellation through a
  shared `context.WithCancel` so siblings exit on the first
  worker that observes `ctx.Err()` (closes #192).
- `flow.MaxFlowCtx` polls `ctx.Err()` inside the augmenting DFS
  at a 1<<12-step granularity; cancellation no longer waits for
  the next BFS-rebuild boundary (closes #193).
- `search.BFSDirectionOpt` no longer carries a writable
  package-global observer hook; the test hook is now an explicit
  `obs func(int, bool)` parameter on the internal `bfsDoCore`
  (closes #194).
- BREAKING: `btree.Index.BulkLoad` signature
  `(values, nodes) → (values, nodes) error`; new sentinel
  `btree.ErrMismatchedLengths` replaces a panic on slice-length
  mismatch (closes #195).
- `errorlint` enabled in `.golangci.yml`; all 63 surfaced
  `fmt.Errorf("%w: ... %v", sentinel, err)` occurrences migrated
  to Go 1.20+ multi-wrap `%w: %w` so `errors.Is` recovers both
  the sentinel and the underlying error (closes #196).
- This release-notes draft (closes #197).

## Breaking changes

- `centrality.WeightedBetweenness` signature changed from
  `([]float64)` to `([]float64, error)`.
- `btree.Index.BulkLoad` signature changed from `(values, nodes)`
  to `(values, nodes) error`. The previous panic on length
  mismatch is now `btree.ErrMismatchedLengths`.
- `store/txn` typed codecs introduce v2 WAL frames tagged with
  `OpRecordV2 = 0xFE`. v1 frames remain decodable; the typed
  constructors (`NewStoreWithCodec`, `NewStoreWithOptions`) are
  new entry points — existing untyped callers see no break, but
  a typed migration is the recommended path.
- `store/snapshot` writes v2 manifests by default. v1 manifests
  still load.

Per `docs/semver.md` these are the only breaking changes in
v1.2.0.

## Deprecated

- `search.EppsteinKShortest` and `EppsteinKShortestCtx`: thin
  `// Deprecated:` wrappers around the renamed
  `KShortestPathsLoopless` / `KShortestPathsLooplessCtx`.
- `store/recovery.OpenString`, `OpenWithCodec`, `OpenWithOptions`:
  thin `// Deprecated:` wrappers around the generic
  `recovery.Open[N, W]` / `OpenCtx[N, W]`.

## Known limitations / deferred work

- Canonical 4-hour soak with continuous FD trace — still tracked
  as a follow-up; the 30-min run is the v1.1.0 / v1.2.0 baseline.
- Multi-concurrency benchmark sweep (1 / 8 / 64 / 256 / 1024
  goroutines) — still not published.
- True Eppstein 1998 (heap-of-heaps via the sidetrack graph
  `D(G)`) — deferred; `KShortestPathsLoopless` is the honest
  best-first implementation.
- Deterministic crash-injection harness in `store/recovery` —
  `csrfile` ships a 200-truncation fuzz; the recovery-side
  equivalent is a follow-up.

## Reproducing the numbers

```bash
# Headline benchmarks.
./scripts/run_headline_bench.sh

# Cross-library comparison (Go + NetworkX + GraphBLAS).
go test ./bench/comparison -bench Comparison -benchmem -count=3
python3 bench/comparison/networkx_baseline.py
make comparison-graphblas

# Reliability soak (default 4 h; -duration overrides).
make soak

# Coverage gate.
make cover-gate
```

All v1.2.0 measurements were taken on an Apple M4 with
`GOMAXPROCS=10`, Go 1.26.3.

## Cutting the tag

The actual `git tag v1.2.0 -m ... && git push --tags` plus
`make release` is intentionally NOT performed by this draft. The
release-management sprint that cuts v1.2.0 is responsible for:

- Promoting `## [Unreleased]` to `## [1.2.0] — <date>` in
  `CHANGELOG.md`.
- Verifying `make cover-gate`, the full CI matrix, and (for a
  production-tier release) the canonical 4-hour soak run.
- Running `goreleaser release --clean` after the tag.
