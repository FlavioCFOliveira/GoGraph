# GoGraph v1.1.0

Released 2026-05-19.

## Headline

v1.1.0 is the first post-1.0 release: six full sprints (8–13) of
correctness, observability, hot-path performance, algorithmic
completeness, and release-hygiene work. Every change ships behind
a property test, a fuzz test, or a measured benchmark. The
correctness defects identified by the v1.0.0 audit are closed.

## Headline numbers (Apple M4)

- `BenchmarkDijkstra_PostWarmup`: 0 allocs/op (zero-alloc steady
  state via `DijkstraInto`).
- `BenchmarkLeiden` at V=1e5: 5.12x faster than v1.0 with 99.96 %
  fewer allocations.
- `BenchmarkBFSDirectionOpt` (V=1M power-law) is 6.08x faster than
  the vanilla top-down BFS once the Beamer beta switch-back kicks
  in.
- `BenchmarkBetweennessParallel` at V=512 is 4.9x faster than the
  serial Brandes on a 10-core machine.
- `ds.UnionFindSlice` is 22.2x faster than the generic map-backed
  variant on a 1 M-node Kruskal workload.
- Cross-library: GoGraph beats NetworkX 3.2.1 on the same 16 K-node
  graph by 178x (BFS), 25x (Dijkstra), 28x (PageRank).

## What's new

### Concurrency contract (Sprint 9)

- Every public blocking API now accepts a `context.Context` and
  honours cancellation and deadlines.
- `goleak.VerifyTestMain` adopted by every package that spawns
  goroutines.

### Observability (Sprint 10)

- Prometheus-style histogram **API hook** in `internal/metrics`: a
  Backend interface, lock-free `atomic.Pointer[Backend]` swap, and
  the `IncCounter` / `ObserveLatency` / `Time` helpers with a
  zero-overhead no-op default. The hook is the interface contract
  for the CLAUDE.md "latency histograms on every public blocking
  API" mandate; **wiring it into individual call-sites across
  `search/`, `store/`, and `graph/io/` is deferred** and tracked
  for a future release. See the `internal/metrics` package doc for
  the rationale (incremental wire-up without further API churn).
- `pprof.SetGoroutineLabels` on every long-lived goroutine.
- `docs/benchmarks/` archive with multi-concurrency-level numbers.
- `govulncheck` and `internal/stress` (concurrency stress under
  `-race`) integrated into CI.
- `csrfile` crash-injection fuzz test for truncation recovery.

### Hot-path optimisation (Sprint 11)

- Zero-allocation primitives: `DijkstraInto`, `BellmanFordInto`,
  `AStarInto`.
- Type-switch dispatch on the per-W `sync.Pool` registry.
- Index-head BFS queue across Brandes, PPR-push, Topo, Dinic,
  Leiden.
- Leiden/LabelPropagation scratch + touched-list pattern replaces
  per-vertex maps.
- BFS-DO inline bitmap frontier + pooled scratch + Beamer beta
  switch-back.
- Iterative DFS for Dinic and Hopcroft-Karp (no goroutine-stack
  growth at LDBC scale).
- Floyd-Warshall column materialisation.
- Hierholzer trail pre-alloc.
- PageRank outdeg float64 → uint32.
- SPFA + SLF Bellman-Ford.
- Yen candidate arena.
- `extern/bfs.go` slices.Sort, `graph/io` Mapper.Walk batch
  resolve, dot writer strconv.FormatInt.
- `ds.UnionFindSlice` for bounded-ID workloads.

### Algorithm completeness (Sprint 12)

- `search.BidirectionalDijkstra` and `BidirectionalDijkstraOn`.
- `flow.EdmondsKarp` (max-flow reference / property baseline).
- `search.KruskalMST` and `search.PrimMST`.
- `search.WCC` (weakly-connected components).
- `search.KCore` (Batagelj-Zaversnik).
- `search.CountTriangles` (degree-ordered node-iterator).
- `search.TransitiveClosure` (bitset oracle).
- `centrality.WeightedBetweenness` (Dijkstra-augmented Brandes).
- `centrality.BetweennessParallel` (work-shards across goroutines).
- `flow.PushRelabelMaxFlow` (FIFO + gap heuristic).
- `search.Diameter` (2-sweep + iFUB).
- `search.HierholzerUndirected`.
- `search.BiBFSOn` (reverse-CSR variant; BiBFS now handles
  directed graphs).
- `search.EppsteinKShortest`.
- `flow.MinCostMaxFlow` (SSP + node potentials).

### Release hygiene (Sprint 13)

- `bench/soak` 4-hour reliability harness with periodic heap /
  goroutine / FD snapshots.
- benchstat regression gate on pull-requests.
- LDBC SF1/SF10 + DIMACS SF1/USA `Benchmark*` functions (the
  large-scale ones skip under `-short`).
- goreleaser pipeline + `.github/workflows/release.yml` + Makefile
  release targets. See `docs/release.md`.
- Cross-library comparison vs NetworkX in `docs/benchmarks/comparison.md`.
- Rapid property tests for Dijkstra triangle inequality,
  TopologicalSort precedence, Tarjan SCC reflexivity, Hopcroft-Karp
  cardinality.
- Fuzz tests for csrfile / graphml / csv.
- t.Run subtest pattern adopted on representative table tests.
- Concurrency-contract godoc clauses on APSP / Matching / TC.

## Breaking changes

Carried over from the Sprint 8 correctness pass (already documented
in the v1.1.0 changelog entry):

- `search.Hungarian` now returns `(Assignment, error)`.
- `centrality.PageRank` now returns `(ranks, iters, error)`.
- `centrality.PersonalisedPushPageRank` now returns `(ranks, error)`.
- `extern.PageRank` now returns `(ranks, iters, error)`.
- `community.Partition.Community[id]` returns `-1` for ghost slots.
- `search.APSP.N` returns the live count, not `MaxNodeID()`.

Per `docs/semver.md` these are the only breaking changes in v1.1.0.

## Deprecated

- `search.JohnsonAPSP`: alias for `DijkstraAPSP`. Will be removed in
  a future major once Bellman-Ford reweighting lands.
- `search.ErrNotUndirected`: never produced any more (BiBFS now
  auto-builds the reverse CSR for directed input).

## Migration

The only signature breaks affect callers of:

```go
// v1.0.0
ranks, iters := centrality.PageRank(c, opts)

// v1.1.0
ranks, iters, err := centrality.PageRank(c, opts)
if err != nil { /* handle */ }
```

Same shape applies to `Hungarian`, `PersonalisedPushPageRank`,
`extern.PageRank`. All other v1.0 APIs are compatible.

## Reproducing the numbers

```bash
# Headline benchmarks.
./scripts/run_headline_bench.sh

# Cross-library comparison.
go test ./bench/comparison -bench Comparison -benchmem -count=3
python3 bench/comparison/networkx_baseline.py

# Soak (4 h).
make soak
```

All numbers above were taken on an Apple M4 with `GOMAXPROCS=10`.
