# Example 18 — Out-of-core pipeline

## What it demonstrates

The complete five-stage out-of-core (Tier 2) flow, end to end: generate a
directed web-link graph as a **CSV edge list**, ingest it through the CSV
reader, freeze the adjacency into an immutable **CSR snapshot**, persist that
snapshot as an on-disk **`csrfile`**, re-open the file by **`mmap`**, and run
**semi-external BFS and PageRank** directly over the mapped region:

```
CSV bytes -> adjacency list -> CSR snapshot -> csrfile (disk) -> mmap -> {BFS, PageRank}
```

Example 05 builds the adjacency directly in memory; this example extends it by
routing the same scale-free graph through the CSV interchange stage first, so
the whole ingest path is exercised. From the `mmap` point on, the graph is
queried straight out of the mapped pages: the semi-external traversal keeps only
vertex-sized auxiliary state (a visited bitset, the rank vector) in RAM while
streaming the adjacency from the mapped file — the out-of-core advantage this
example measures.

## Domain / scenario

A directed, scale-free **hyperlink graph** (a miniature web / citation
network), generated by **Price's model** (de Solla Price, 1965/1976) — the
directed preferential-attachment process that is the canonical domain for
PageRank. Pages arrive in id order; each arriving page emits `m` citation
out-links to distinct earlier pages, choosing each target with probability
proportional to its current in-degree plus a small attractiveness constant `a`
(in-degree exponent `gamma_in = 2 + a/m`). A few early-arriving pages accrue
most inbound links and become high-PageRank **authorities**, while the bulk hold
near-uniform low rank, so the top-k highest-PageRank ids are a meaningful,
discriminating fact.

Pure citation edges always point newer → older, which makes the graph a shallow
DAG (forward BFS from an old page would reach almost nothing). To give the BFS a
meaningful, stable reachable count, each arriving page also receives, with
probability `nav-frac`, one **navigation link** from a uniformly random earlier
page (older → newer); these forward links build long forward paths without
disturbing the authority tail. The BFS is seeded from a fixed mid-graph
**portal** page (id `N/4`, never the oldest page, which is a pure sink), from
which roughly 54% of the graph is reachable at the default scale.

The CSV reader interns each page name to a compact `NodeID`. The `csrfile`
preserves `NodeID`s across the `mmap` boundary but **drops the name → `NodeID`
mapping**, so the BFS needs a known numeric seed: the example captures the
portal page's `NodeID` from the reader's mapper before the adjacency (and its
mapper) is discarded — exactly the pattern a real out-of-core consumer follows.
Note the captured seed id (`1280`) is **not** the generator id (`1000`), because
the CSV interns nodes in edge-appearance order, not numeric order.

## How to run

```sh
go run ./examples/18_oocore_pipeline                       # small deterministic default
go run ./examples/18_oocore_pipeline -nodes 2000000 -out-degree 8  # observable-scale run
```

## Scale and flags

| Flag | Meaning | Default | Representative large value |
|---|---|---|---|
| `-nodes` | number of pages in the web graph | `4000` | `2000000` |
| `-out-degree` | citation out-links per arriving page (Price's `m`) | `4` | `8` |
| `-attractiveness` | additive attractiveness constant (Price's `a`; `> 0`) | `1` | `2` |
| `-nav-frac` | probability an arriving page gets a forward navigation link | `0.5` | `0.5` |
| `-top-k` | number of highest-PageRank node ids to report | `10` | `10` |
| `-seed` | RNG seed (fixes the deterministic data shape) | `7` | any |

The default builds, persists, mmaps, and queries in milliseconds — the size the
regression test pins. The large invocation is where the out-of-core footprint
becomes interesting: at two million pages and eight out-links each the on-disk
`csrfile` is on the order of ~270 MiB while the resident rank vector is only
~16 MiB.

## Evidence it collects

This is an **out-of-core** example, so it reports the dimensions that matter for
that subject (per the evidence taxonomy in
[`docs/examples-standard.md`](../../docs/examples-standard.md)):

- **Per-stage wall-clock** for every stage of the pipeline — CSV ingest, CSR
  build, csrfile write, mmap open, BFS, and PageRank — so a reader can see where
  time goes as the graph scales.
- **On-disk size vs. resident heap**: `disk.csrfile` (the adjacency footprint on
  disk) against `mem.rank_vector` / `mem.heap_alloc` (the in-RAM working set),
  with their ratio `ooc.disk_over_rank_vector`. This is the headline out-of-core
  advantage — scale `-nodes` up and the ratio widens because the adjacency stays
  on disk while only the vertex-sized rank vector grows in RAM.

The deterministic facts are the correctness anchor: `csv.edges == csr.size`
proves the pipeline neither dropped nor duplicated an edge across the
`CSV -> CSR -> csrfile` boundaries; `bfs.reachable` and the PageRank top-k pin
the shape and the analytics results for the fixed seed.

## Key APIs

- `graph/io/csv.ReadIntoCtx` / `csv.DefaultOptions` — ingest an edge-list CSV into an in-memory adjacency list and report the edge count.
- `graph/adjlist.AdjList.Mapper` + `graph.Mapper.Lookup` — capture the seed page's `NodeID` before the mapper is discarded at the mmap boundary.
- `graph/csr.BuildFromAdjList` / `csr.CSR.Order` / `csr.CSR.Size` — freeze the adjacency into an immutable Tier 1 CSR snapshot.
- `store/csrfile.WriteToFile` — atomically persist the CSR snapshot as a Tier 2 memory-mapped file.
- `store/csrfile.Open` / `csrfile.Reader.SetHint` / `csrfile.AccessSequential` — re-open the file via `mmap` and hint a sequential access pattern.
- `search/extern.BFSCtx` — semi-external breadth-first traversal over the mapped region from a seed `NodeID`.
- `search/extern.PageRankCtx` / `extern.DefaultPageRankOptions` — semi-external PageRank over the mapped region.
